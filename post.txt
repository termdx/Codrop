Launching Codrop: a Dropbox for developers.

I switch between machines constantly: laptop, desktop, remote box, cloud agents. The code is “the same project,” but the working state never is.

One machine has the half-finished refactor.
Another has the `.env` that makes it run.
A third has the fix I made before bed and forgot to commit.

The usual answers are all a little broken:

- `git commit` work that isn’t ready to commit
- `git stash` and hope you remember where it lives
- push temporary branches
- copy files manually
- send yourself a Slack message
- rebuild context from scratch on the next machine

Git is excellent at recording intentional history: commits, PRs, review, releases.

But Git was never meant to answer: “I closed my laptop and opened my desktop—where is the exact state of my work?”

That’s what Codrop is for.

Codrop watches a folder and continuously syncs its current working state across your devices and cloud agents.

Not just committed code. The actual folder you are working in:

- uncommitted edits
- new files
- local configuration
- generated state
- half-written features
- the tiny changes that are too early, too messy, or too private for a commit

It runs over encrypted peer-to-peer connections. Your devices authenticate with public keys, not IP addresses. It works across LANs, NAT, and restrictive Wi-Fi—connecting directly where possible and relaying when needed.

The workflow is deliberately boring:

```bash
# on machine A
cd ~/code/my-project
codrop id

# on machine B
cd ~/code/my-project
codrop pair <machine-a-id>
codrop run
```

Pair the other side once, run Codrop, and your folder stays in sync.

No central server to manage.
No cloud account.
No “which version did I push?”
No copying a zip file named `project-final-final-2`.

Codrop is not “Git, but faster.”

Use Git for durable history and collaboration.
Use Codrop for the living, uncommitted state between commits.

It content-addresses changes, transfers only what changed, preserves file modes and symlinks, and uses vector clocks to order concurrent edits. If two machines really do collide, it preserves the conflict instead of silently destroying work.

It also respects ignores, so you can keep things local:

```bash
codrop ignore .env
```

Codrop is Unix-only for now, and it’s open source under MIT.

Install it with Homebrew:

```bash
brew install termdx/tap/codrop
```

Or grab the latest release:

https://github.com/termdx/Codrop

The goal is simple: your code should be where you are—before you even think about it.
